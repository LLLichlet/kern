# Security Policy

Kern is a personal project maintained actively, but without a dedicated security
response team. Please use the guidance below to report security issues
privately and with enough detail to reproduce them.

## Supported Versions

Security fixes are considered for the current `main` branch and the latest
published release. Older experimental releases are not guaranteed to receive
backported fixes.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability before it has
been triaged.

Preferred reporting path:

- Use GitHub private vulnerability reporting if it is available for this
  repository.
- Email softfault at `givenyoung46@gmail.com`.
- If private vulnerability reporting is not available, open a minimal public
  issue asking for a private contact path without including exploit details.

Useful reports include:

- The affected component, such as `kernc`, `craft`, `kern-lsp`, `library/`,
  installers, or editor support.
- The affected version or commit.
- The operating system and relevant configuration.
- Reproduction steps or a proof of concept.
- The expected impact and any known mitigations.

## Disclosure

Please allow time for triage and a fix before publishing details. Response and
fix timing depends on project availability, issue complexity, and release
readiness, so this project does not promise a fixed response window.

After a fix is available, public disclosure can be coordinated through the
release notes, advisory text, or a public issue.
