# Kern 0.6.4 Release Notes

`0.6.4` is a tooling-focused release centered on making Kern materially easier
to edit with language-aware tooling.

## Highlights

- `kern-lsp` now has stronger completion behavior, more accurate signature help
  argument tracking, and sharper semantic token classification.
- The repository now includes a first-party preview VS Code extension under
  `editors/vscode/`.
- Native Kern syntax highlighting in VS Code has been pushed beyond a generic
  Rust-style fallback, with explicit support for Kern-specific constructs such
  as:
  - intrinsics like `@asm` and `@trap`
  - `Type.{ ... }` data constructors
  - enum variants such as `.Some`
  - pointer/slice postfix operators like `.*`, `.&`, `..&`, `.[`, and `..[`
  - closure capture syntax `.[ ... ]`
  - generic type declarations and `impl` headers

## Toolchain Packaging

Release archives now ship:

- `kernc`
- `craft`
- `kern-lsp`
- `library/std`

This makes editor integration possible from installed toolchains without
requiring a repository-local build.

## Status

- VS Code integration in `0.6.4` should be treated as a first-party preview.
- The broader LSP maturity push remains planned to continue through `0.6.5`.
