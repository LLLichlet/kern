# 01. 快速开始

[English](../en/01-quick-start.md) | 简体中文

## 安装工具链

Linux 和 macOS:

```sh
curl -sSf https://raw.githubusercontent.com/kern-project/kern/main/install.sh | bash
```

Windows PowerShell:

```powershell
powershell -Command "Set-ExecutionPolicy Bypass -Scope Process -Force; Invoke-Expression (Invoke-WebRequest -Uri https://raw.githubusercontent.com/kern-project/kern/main/install.ps1 -UseBasicParsing).Content"
```

安装器会把 SDK 放到 Unix 的 `~/.kern` 或 Windows 的 `%USERPROFILE%\.kern` 下，并检查 `kernc`、`craft`、`kern-lsp` 能否启动。

## 创建第一个包

```sh
mkdir hello
cd hello
craft init
```

`craft init` 会创建一个最小包。当前版本通常会生成这些项目文件：

```text
Craft.toml
Craft.lock
.gitignore
src/main.rn
```

你也可能看到 `.craft/`。这是 `craft` 的本地派生状态目录，用来放锁、缓存、构建产物和分析状态；它不是需要手写维护的源码。

把 `src/main.rn` 改成：

```kern
use std.io;

fn main() i32 {
    "Hello, Kern!".println();
    return 0;
}
```

然后运行：

```sh
craft run
```

这段程序只用了三个概念：

- `use std.io;` 引入标准库里的输出能力。Kern 不会自动把标准库名字放进当前文件。
- `fn main() i32` 定义可执行程序入口。返回的 `i32` 会作为程序退出码，`0` 通常表示成功。
- `.println()` 把字节文本写到标准输出，并在末尾换行。

如果要把变量放进输出文本，可以使用格式化：

```kern
use std.io;

fn main() i32 {
    let name = "Kern";
    "Hello, {}!".fmt(.{name}).println();
    return 0;
}
```

这里的 `.{name}` 是一个轻量的参数聚合值。现在先把它理解成“传给格式化模板的一组参数”即可；后面的章节会讲结构体、匿名聚合和切片。

## 常用的 `craft` 命令

刚开始写 Kern 程序时，先记住这三个命令就够了：

```sh
craft check
craft build
craft run
```

它们的区别是：

- `craft check`：检查代码能否通过解析、类型检查和语义分析，通常比完整构建更快。
- `craft build`：构建当前包的默认目标。
- `craft run`：构建并运行当前包的默认可执行目标。

测试、示例、release profile、清理缓存和跨包选择会在后面的包管理章节展开。如果你在项目根目录下，有 `Craft.toml`，通常就从 `craft run` 开始。

仓库里的 `examples/` 本身也是一个 Craft 项目。在kern仓库根目录下，你可以这样运行一个示例：

```sh
craft run -p examples --example hello_world
```

也可以构建所有示例：

```sh
craft build -p examples --examples
```

## `craft` 和 `kernc` 的分工

平时写项目，用 `craft`。

`craft` 会读取 `Craft.toml`，决定要构建哪个包、哪个目标、哪些依赖和哪些生成步骤，然后调用编译器完成实际工作。

`kernc` 是底层编译和链接驱动。你需要精确控制单个 `.rn` 文件、对象文件、LLVM IR、模块路径或链接参数时，才会直接使用它。例如：

```sh
kernc --runtime-entry rt --library-bundle std examples/hello_world.rn -o hello
./hello
```

刚开始学习时可以先跳过 `kernc`。本教程会优先使用 `craft`，等进入 freestanding 和底层链接时再回到这些参数。完整参考见 [`craft.md`](../../craft.md) 和 [`kernc.md`](../../kernc.md)。
