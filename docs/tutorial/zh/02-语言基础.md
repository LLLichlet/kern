# 02. 语言基础

[English](../en/02-language-basics.md) | 简体中文

## 一个小程序

```kern
use std.io;

fn add(a: i32, b: i32) i32 {
    return a + b;
}

fn main() i32 {
    let answer = add(20, 22);
    "answer = {}".fmt(.{answer}).println();
    return 0;
}
```

Kern 的函数声明写成：

```kern
fn name(arg: Type) ReturnType {
    ...
}
```

没有有意义返回值时使用 `void`：

```kern
fn say(text: &[u8]) void {
    text.println();
}
```

## 绑定与 mutability

局部绑定使用 `let`：

```kern
let x = 10;
let mut y = 20;
y += 1;
```

Kern 的一个重要写法是“类型提供者 + 初始化体”：

```kern
let mut value = i32.{10};
value = 11;
```

`i32.{10}` 的意思是：用 `i32` 作为明确的类型来源来构造值 `10`。这不是类型后缀，也不是从一个无类型值做隐式转换；它和 Kern 其他初始化形式使用同一套语法：

```kern
let byte = u8.{0xff};
let size = usize.{1024};

struct Point {
    x: i32,
    y: i32,
};

let p = Point.{ x: 1, y: 2 };
let bytes = [4]u8.{ 1, 2, 3, 4 };
```

当上下文已经足够清楚时，可以省略类型提供者：

```kern
let mut i = 0;
while (i < 10) {
    i += 1;
}
```

当宽度、ABI、位模式或指针相邻逻辑很重要时，保留类型提供者会更清楚：

```kern
let mask = u8.{0xff};
let null = usize.{0};
```

`mut` 修饰的是存储位置，不是类型本身。`let mut value` 表示这个本地存储可以被重新赋值或原地修改；它不会把所有从它派生出的指针或切片自动升级成可写权限。

```kern
let mut value = i32.{10};

let ptr = value..&;
ptr.* = 12;
```

这里 `value..&` 取得 `&mut i32`，`ptr.*` 通过指针写回。Kern 不做自动解引用，因此指针读写使用显式的 `.*`。

## 基本类型

常见内建类型包括：

- 整数：`i8`、`i16`、`i32`、`i64`、`i128`，以及 `u8`、`u16`、`u32`、`u64`、`u128`。
- 指针宽度整数：`usize`、`isize`。
- 浮点：`f32`、`f64`。
- 布尔：`bool`。
- 无返回：`!`。
- 空值类型：`void`。

Kern 有上下文类型推断。很多时候可以写 `let x = 0;`，让使用点决定类型；当类型本身是逻辑的一部分时，推荐保留类型提供者：

```kern
let byte = u8.{0xff};
let max = usize.{1024};
```

## 字符串与字节

Kern 的字符串字面量不是隐式 NUL 结尾的 C 字符串。`"Hello"` 是 `[5]u8`，也就是一个固定长度为 5 字节的数组。

数组类型写成 `[N]T`：

```kern
let bytes = [4]u8.{ b'k', b'e', b'r', b'n' };
```

切片类型写成 `&[T]` 或 `&mut [T]`。切片不是拥有数据的数组，而是一个视图：它指向一段连续元素，并携带长度。

```kern
let view = bytes.&[1 .. 4];
```

这里 `view` 的类型是 `&[u8]`，表示从 `bytes` 中取出的只读字节切片。可写切片使用 `..&`：

```kern
let mut data = [4]u8.{ 1, 2, 3, 4 };
let head = data..&[0 .. 2];
```

传给需要 `&[u8]` 的函数时，数组可以在边界处自然转换成切片：

```kern
fn len(text: &[u8]) usize {
    return #text;
}

fn main() i32 {
    let n = len("kern");
    "len = {}".fmt(.{n}).println();
    return 0;
}
```

`#` 用来提取数组、切片等值携带的元数据。对数组和切片来说，它就是长度。

## 结构体与方法

```kern
struct Point {
    x: i32,
    y: i32,
};

impl &mut Point {
    fn move_by(dx: i32, dy: i32) void {
        self.x += dx;
        self.y += dy;
    }
}

fn main() i32 {
    let mut p = Point.{ x: 1, y: 2 };
    p..&.move_by(3, 4);
    "({}, {})".fmt(.{p.x, p.y}).println();
    return 0;
}
```

`impl` 写在具体值类型上。Kern 按值来建模：`Point` 是值，`&Point`、`&mut Point` 这样的指针也是值，`&[u8]` 这样的切片同样也是值。它们是不同的类型，因此可以拥有不同的方法集合。只读方法通常放在 `impl &T` 上；会修改接收者的方法放在 `impl &mut T` 上。

可以把 `impl &mut Point` 理解成：给类型 `&mut Point` 挂上一组函数，并在函数体里自动提供一个隐藏的 `self: &mut Point`。调用：

```kern
p..&.move_by(3, 4);
```

等价于先取得 `&mut Point`，再用方法语法调用这个接收者上的函数。Kern 不做自动解引用；方法放在哪个接收者类型上，是 API 设计的一部分。

指针类型、切片类型、函数对象类型等都是普通的具体值类型，所以都可以有自己的方法。例如标准库会给 `&[T]`、`&mut [T]`、`&[u8]` 这样的切片类型提供方法；你自己的类型也可以按需要选择接收者：

```kern
impl &mut Point {
    fn clear() void {
        self.x = 0;
        self.y = 0;
    }
}
```

## 控制流基础

Kern 是表达式驱动的语言。`if` 的语法是：

```kern
let max = if (a > b) a else b;
```

分支本身也是表达式。需要多步逻辑时就使用块：

```kern
let max = if (a > b) {
    a
} else {
    b
};
```

同样，`if` 后面不需要 C 那种“单语句特判”。它只是接收一个表达式作为分支体：

```kern
if (len == 0) return 0;
```

`while` 也是 `while (cond) expr`。循环体通常写成块，因为大多数循环要执行多步操作：

```kern
let mut i = 0;
let mut total = 0;
while (i < 5) {
    total += i;
    i += 1;
}
```

如果循环体只有一个表达式，也可以直接写：

```kern
while (ready()) tick();
```

循环表达式通常作为 `void` 使用；无法正常结束的循环可以表现为发散计算。

迭代器可以用于 `for`：

```kern
use base.coll.range;

let mut sum = 0;
for (i: range(1, 4)) {
    sum += i;
}
```

`for` 会把迭代器保存成内部可变状态，并反复调用 `next()`。这仍然是普通语言机制的组合，不依赖隐藏运行时。下一章会用 `enum`、`match`、`?T` 和 `T!E` 继续展开 Kern 的状态表达和错误处理方式。
