# 03. 数据与控制流

[English](../en/03-data-and-control-flow.md) | 简体中文

## struct

```kern
struct Point {
    x: i32,
    y: i32,
};

fn translate(point: Point, dx: i32, dy: i32) Point {
    return Point.{
        x: point.x + dx,
        y: point.y + dy,
    };
}
```

构造结构体时可以写完整类型：

```kern
let p = Point.{ x: 1, y: 2 };
```

也可以在返回值或参数位置由上下文提供类型：

```kern
return .{ x: point.x + dx, y: point.y + dy };
```

这行能省略 `Point`，是因为 `translate` 的返回类型已经写明是 `Point`。Kern 会把 `.{ ... }` 当成“由上下文提供目标类型的初始化体”。这种省略只应该在目标类型很近、很明确时使用；刚开始学习时，先写完整的 `Point.{ ... }` 更容易读。

普通 `struct` 允许编译器为了更紧凑的布局重排字段。需要 C ABI、硬件寄存器描述或固定源代码字段顺序时，用 `extern struct`：

```kern
extern struct Header {
    tag: u8,
    value: u64,
};
```

## enum 与 match

Kern 用 `enum` 表达一组互斥状态。最简单的 enum 只有名字：

```kern
enum Color: u8 {
    Red = 0,
    Green = 1,
    Blue = 2,
};
```

对 `Color` 这种简单 enum 来说，`Color: u8` 指定底层判别值的整数类型。`Red = 0` 这样的写法指定具体判别值；没有显式写值的 variant 会按顺序递增。没有写 `: u8` 这类 backing type 时，简单 enum 默认使用 `u32`。

enum 也可以携带数据：

```kern
enum ParseResult {
    Number: i32,
    Missing,
};
```

这里 `ParseResult` 有两个 variant：

- `Number: i32` 携带一个 `i32`。
- `Missing` 不携带数据。

构造 enum 值时，先看完整写法：

```kern
let ok = ParseResult.{ Number: 42 };
let missing = ParseResult.Missing;
```

如果目标类型已经由上下文提供，也可以省略类型名：

```kern
fn parse_value() ParseResult {
    return .{ Number: 42 };
}
```

这和上一节的 `return .{ x: ..., y: ... };` 是同一个规则：`.{ ... }` 本身没有写出类型，类型来自附近的返回类型、参数类型或变量声明。

## match 与模式

`match` 根据一个值的形状选择分支。它不只用于 enum；只要能写出模式，就可以用 `match`。最简单的模式可以是字面量或 `_`：

```kern
fn code_score(code: u8) i32 {
    return match (code) {
        0 => 10,
        1 => 20,
        _ => -1,
    };
}
```

`_` 表示“其他情况”。每个 arm 左边是模式，右边是表达式：

```kern
match (value) {
    pattern => expr,
    pattern => expr,
}
```

一个 arm 可以列出多个模式，也可以使用范围模式：

```kern
fn class_of(byte: u8) i32 {
    return match (byte) {
        b'0' ..= b'9' => 1,
        b'a' ..= b'z', b'A' ..= b'Z' => 2,
        b'_', b'-' => 3,
        _ => 0,
    };
}
```

`a .. b` 表示不包含右端点的范围，`a ..= b` 表示包含右端点的范围。多个模式用逗号写在同一个 arm 左侧，表示这些情况共享同一个结果表达式。

普通 value pattern 使用和 `==` 相同的相等性能力。也就是说，只要被匹配值的类型实现了对应的 `Eq[...]`，就可以把字符串、切片或自定义值直接写在 arm 左侧：

```kern
fn command_id(command: &[u8]) i32 {
    return match (command) {
        "run" => 1,
        "test" => 2,
        _ => 0,
    };
}
```

对这种开放值域，通常仍然要保留 `_` arm；只有 enum、bool、整数范围等编译器能够证明覆盖完整的模式，才可以省略兜底分支。

对 enum 来说，模式可以检查当前是哪一个 variant，并取出 payload：

```kern
fn describe(result: ParseResult) void {
    match (result) {
        ParseResult.{ Number: value } => "number = {}".fmt(.{value}).println(),
        ParseResult.Missing => "missing".println(),
    }
}
```

`ParseResult.{ Number: value }` 表示：如果 `result` 是 `Number`，就把它携带的 `i32` 绑定到局部名字 `value`。

绑定也可以是可变的：

```kern
ParseResult.{ Number: mut value } => {
    value += 1;
    "next = {}".fmt(.{value}).println();
}
```

`ParseResult.Missing` 表示：匹配不携带数据的 `Missing` variant。

因为 `match (result)` 已经说明被匹配的值是 `ParseResult`，所以上面也可以写成更短的形式：

```kern
fn describe(result: ParseResult) void {
    match (result) {
        .{ Number: value } => "number = {}".fmt(.{value}).println(),
        .Missing => "missing".println(),
    }
}
```

`match` 也可以直接返回值：

```kern
fn score(result: ParseResult) i32 {
    return match (result) {
        .{ Number: value } => value,
        .Missing => 0,
    };
}
```

如果 enum 有多个 variant，`match` 必须覆盖所有 variant。这样后续给 enum 增加新状态时，编译器可以指出哪些分支还没有处理。

模式还可以嵌套。先看一个较小的例子：

```kern
enum Bit {
    Zero,
    One,
};

enum Leaf {
    Empty,
    Full: Bit,
};

fn leaf_score(leaf: Leaf) i32 {
    return match (leaf) {
        Leaf.Empty => 0,
        Leaf.{ Full: Bit.Zero } => 1,
        Leaf.{ Full: Bit.One } => 2,
    };
}
```

`Leaf.{ Full: Bit.One }` 会先匹配 `Leaf.Full`，再继续检查它携带的 `Bit` 是否是 `Bit.One`。

如果外层上下文已经明确，也可以把类型省略掉：

```kern
fn leaf_score(leaf: Leaf) i32 {
    return match (leaf) {
        .Empty => 0,
        .{ Full: .Zero } => 1,
        .{ Full: .One } => 2,
    };
}
```

本章后面的例子会经常使用这种省略形式，但规则始终相同：省略掉的类型必须能从附近上下文恢复出来。

## Option：`?T`

`?T` 是内建 enum 家族，表示“有值或无值”。它可以理解成语言内建的泛型 enum 形态：

```kern
enum Option[T] {
    Some: T,
    None,
};
```

区别是 `?T` 是语言级类型写法，地位和 `i32`、`&T`、`[N]T` 一样直接。它不是 nullable pointer，也没有特殊的隐藏 ABI。

```kern
fn first_digit(text: &[u8]) ?u8 {
    let mut i = 0;
    while (i < #text) {
        let byte = text.[i];
        if (byte >= b'0' and byte <= b'9') {
            return .{ Some: byte };
        }
        i += 1;
    }
    return .None;
}
```

使用时通常 `match`。可以把模式匹配先理解成“带解包能力的分支判断”：普通 `if` 只能判断条件真假，而 `match` 的每个分支会同时说明“值应该长什么样”以及“如果长这样，要把哪些内部数据取出来”。

```kern
match (first_digit("abc3")) {
    .{ Some: byte } => "digit = {}".fmt(.{byte}).println(),
    .None => "no digit".println(),
}
```

如果只关心有没有值，可以用 `is_some()` / `is_none()`：

```kern
if (first_digit("abc3").is_some()) {
    "found a digit".println();
}
```

如果要把里面的值变形，可以用 `map`：

```kern
let digit_value = first_digit("abc3").map([](byte: u8) u8 {
    return byte - b'0';
});
```

这里 `digit_value` 的类型是 `?u8`。输入是 `Some(b'3')` 时得到 `Some(3)`；输入是 `None` 时仍然是 `None`。

如果要把“缺少值”变成一个带类型的错误，可以用 `ok_or`：

```kern
enum ParseError {
    MissingDigit,
};

let digit = first_digit("abc").ok_or(ParseError.MissingDigit);
```

这里 `digit` 的类型是 `u8!ParseError`。

## Result：`T!E`

`T!E` 表示“成功值 `T` 或错误 `E`”。Kern 没有异常；失败是普通值，也是普通控制流。

```kern
enum Error {
    Empty,
};

fn first(text: &[u8]) u8!Error {
    if (#text == 0) {
        return .{ Err: .Empty };
    }
    return .{ Ok: text.[0] };
}
```

可以手写 `match`：

```kern
match (first("kern")) {
    .{ Ok: byte } => "first = {}".fmt(.{byte}).println(),
    .{ Err: .Empty } => "error: empty input".println(),
}
```

当分支里要做实质工作时，直接写 `match` 最清楚。只有在控制流只是“成功就继续，失败就返回”时，才适合使用更短的形式。

`let else` 是 Kern 里很常用的控制流形式。它不是 `match` 的替代品，而是专门表达“我只想让某一个模式继续往下走；其他情况必须离开当前路径”。

最短形式是：

```kern
let .{ Ok: byte } = first(text) else return .{ Err: .Empty };
```

这行的意思是：

- 如果 `first(text)` 是 `Ok`，把 payload 绑定成 `byte`，后面的代码可以继续使用它。
- 如果不是 `Ok`，执行 `else` 后面的表达式。这里是 `return`，所以当前函数直接结束。

如果失败情况需要区分，可以把 `else` 写成 failure-arm block：

```kern
fn first_or_error(text: &[u8]) u8!Error {
    let .{ Ok: byte } = first(text)
        else {
            .{ Err: err } => return .{ Err: err },
        };

    return .{ Ok: byte };
}
```

`else` 里的 arm 匹配的是原来的失败值；绑定只在对应 arm 内有效。`let else` 成功绑定出来的名字则进入外层作用域。

如果失败时只需要换成另一个错误类型，通常可以用方法组合让代码更短：

```kern
enum OuterError {
    Parse: Error,
};

fn first_wrapped(text: &[u8]) u8!OuterError {
    let byte = first(text)
        .map_err([](err: Error) OuterError { return .{ Parse: err }; })
        .!;
    return .{ Ok: byte };
}
```

`.!` 用在返回类型同样是 `T!E` 的函数里，表示“取出 `Ok`，如果是 `Err` 就从当前函数返回这个错误”：

```kern
fn first_or_error(text: &[u8]) u8!Error {
    let byte = first(text).!;
    return .{ Ok: byte };
}
```

`.?` 是 `?T` 的对应写法：取出 `Some`，如果是 `None` 就从当前函数返回 `None`。

```kern
fn first_digit_plus_one(text: &[u8]) ?u8 {
    let digit = first_digit(text).?;
    return .{ Some: digit + 1 };
}
```

## 匿名聚合

Kern 支持匿名 struct、union 和 enum。它们适合局部数据组织、结构化边界和轻量的 duck typing：

```kern
fn sum_pair(pair: struct { y: i32, x: i32 }) i32 {
    return pair.x + pair.y;
}

fn read_word(word: union { bytes: [4]u8, int: i32 }) i32 {
    return word.int;
}
```

匿名 struct 是结构化类型；字段顺序不会像命名类型那样形成不同类型。下面是一个完整例子：

```kern
use std.io;

struct Pair {
    x: i32,
    y: i32,
};

fn sum_pair(pair: struct { y: i32, x: i32 }) i32 {
    return pair.x + pair.y;
}

fn main() i32 {
    let pair = Pair.{ x: 4, y: 5 };
    let total = sum_pair(pair);
    "total = {}".fmt(.{total}).println();
    return 0;
}
```

`sum_pair` 接收的是匿名 struct 类型 `struct { y: i32, x: i32 }`。`Pair` 是命名 struct；在函数参数这样的明确边界处，Kern 可以把具名 struct 按字段形状转换到兼容的匿名 struct。

这个方向是有意设计的：具名类型更具体，匿名类型更结构化。匿名 struct 适合描述“我只需要这些字段”的局部边界；真正表达领域概念、维护 API 身份时，仍然应该使用具名 `struct`。

匿名 union 适合临时表达一组共享存储的不同视图：

```kern
fn read_word(word: union { bytes: [4]u8, int: i32 }) i32 {
    return word.int;
}

let word = union { bytes: [4]u8, int: i32 }.{ int: 11 };
let value = read_word(word);
```

匿名 enum 适合局部状态集合：

```kern
fn classify(state: enum: u32 { Off = 0, On = 1, Error: i32 }) i32 {
    return match (state) {
        .Off => 0,
        .On => 1,
        .{ Error: code } => code,
    };
}
```

匿名 enum 也可以指定底层判别类型和显式判别值，写法和命名 enum 一样：

```kern
let state = enum: u32 { Off = 0, On = 1, Error: i32 }.{ Error: 13 };
```

需要固定 C ABI 布局时，匿名 struct 同样可以写成 `extern struct { ... }`。更完整的布局示例见 [`examples/anonymous_aggregates.rn`](../../../examples/anonymous_aggregates.rn)。
