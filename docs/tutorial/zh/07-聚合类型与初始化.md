# 07. 聚合类型与初始化

[English](../en/07-aggregates-and-initialization.md) | 简体中文

前几章已经用过 `struct`、匿名结构体、数组、enum 和 union。本章把这些聚合类型的初始化规则集中讲清楚，尤其是 Kern 很常用的“结构体默认字段”模式。

## struct 可以提供默认字段

Kern 的 struct 字段可以带默认值：

```kern
struct Config {
    port: u16 = 8080,
    retries: u8 = 3,
    verbose: bool = false,
};
```

初始化时，没有默认值的字段必须显式提供；有默认值的字段可以省略：

```kern
let default = Config.{};
let custom = Config.{ port: 9000 };
```

字段默认值是表达式，不只是字面量。它会在 struct 声明处先按字段类型做类型检查；当某个初始化表达式省略这个字段时，编译器会在那个构造点使用这段默认表达式来生成字段值。

```kern
struct Limits {
    max_open: usize = {
        static fallback = 256usize;
        fallback
    },
};

let limits = Limits.{};
```

这意味着默认表达式按普通 Kern 表达式规则工作：可以用块表达式，也可以在块里写局部 `static`。不要把它理解成“声明时计算一次然后缓存到字段里”；每个省略字段的初始化点都会使用这个默认表达式。若默认表达式内部声明了 `static`，共享的是那个显式 `static`，不是字段默认机制本身。

这让 struct 可以作为一种“默认参数提供者”。Kern 没有函数默认参数这一层语法时，很多 API 会把可选项集中成一个配置结构：

```kern
fn connect(options: Config) i32 {
    if (options.verbose) {
        return options.port as i32 + options.retries as i32;
    }
    return options.port as i32;
}

let code = connect(.{ port: 9000 });
```

这里 `.{ port: 9000 }` 是上下文初始化：`connect` 的参数类型已经是 `Config`，所以初始化表达式可以省略类型名。刚开始写教程代码时，推荐先写完整的 `Config.{ ... }`，等上下文很清楚时再省略。

## 字段 pun

typed struct 初始化里，如果局部绑定名和字段名一致，可以使用 field pun：

```kern
struct Pair {
    x: i32,
    y: i32,
};

let x = 4i32;
let y = 5i32;

let pair = Pair.{ x, y };
```

这等价于：

```kern
let pair = Pair.{ x: x, y: y };
```

field pun 只是一种局部简写。跨模块、跨 API 边界或字段较多时，显式写 `field: value` 往往更容易读。

## `undef` 必须显式写出来

如果字段没有默认值，省略它是编译错误。确实需要未初始化存储时，要显式写 `undef`：

```kern
struct Packet {
    len: usize,
    data: [1500]u8,
};

let packet = Packet.{
    len: 0,
    data: undef,
};
```

这条规则很重要：Kern 不会把“忘了填字段”和“故意留未初始化”混在一起。

## 原生布局与 `extern struct`

普通 Kern struct 使用原生布局。编译器可以为了减少 padding 调整物理字段顺序，所以源码字段顺序不是 ABI 承诺。

```kern
struct NativeLayout {
    tag: u8,
    value: u64,
    flag: u16,
};
```

如果类型要和 C、硬件表、boot protocol 或汇编边界共享布局，使用 `extern struct`：

```kern
extern struct CLayout {
    tag: u8,
    value: u64,
    flag: u16,
};
```

这不是性能注解，而是 ABI 契约。普通应用内部数据优先用原生 struct；边界数据才用 `extern struct`。

## anonymous struct

匿名结构体是结构类型，适合轻量分组、临时返回值和闭包状态：

```kern
fn bounds() struct { start: usize, end: usize } {
    return .{ start: 2, end: 8 };
}
```

匿名结构体按字段集合和字段类型等价，不按某个声明名字等价。命名 struct 则是名义类型，即使字段一样，也仍然是不同类型。

匿名结构体也可以写成 `extern struct { ... }`，用于 inline 表达 C ABI 边界：

```kern
extern {
    fn consume(header: &extern struct { tag: u8, len: u32 }) void;
}
```

## union 和 enum 初始化

union 没有 active-field 跟踪，也没有默认字段。初始化时必须明确选择一个字段：

```kern
let word = union { bytes: [4]u8, int: i32 }.{ int: 11 };
```

payload enum 的初始化也显式选择 variant：

```kern
let state = enum: u32 {
    Off = 0,
    On = 1,
    Error: i32,
}.{ Error: 13 };
```

Kern 的 enum 是强独立类型。外部整数不能通过 `as` 直接 cast 成 enum，语法和类型检查会杜绝这种构造方式。要把硬件、文件格式或网络协议里的整数转换成 enum，先用 `match` 做清洗：

```kern
enum Color: u8 {
    Red = 0,
    Green,
    Blue,
};

fn color_from_byte(raw: u8) Color {
    return match (raw) {
        0 => Color.Red,
        1 => Color.Green,
        2 => Color.Blue,
        _ => Color.Red,
    };
}
```

## 类型信息 intrinsic

聚合类型经常和类型信息 intrinsic 一起用：

```kern
let size = @sizeOf[CLayout]();
let align = @alignOf[CLayout]();
```

`@sizeOf[T]()` 和 `@alignOf[T]()` 在编译期求值。它们常用于 ABI 检查、allocator layout、静态断言和低层内存计算。

闭包这种匿名类型还会用到 `@typeOf(expr)`。第 09 章会专门讲闭包状态和 `@typeOf` 的关系。
