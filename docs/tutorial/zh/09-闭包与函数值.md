# 09. 闭包与函数值

[English](../en/09-closures-and-function-values.md) | 简体中文

Kern 的闭包不是隐藏运行时对象。它由两层组成：一个具体的匿名状态值，以及一个可调用的函数入口。理解这两层，才能正确判断闭包什么时候能传递、什么时候会逃逸、什么时候需要显式分配。

## 函数指针

没有捕获状态的函数值可以用普通函数指针表示：

```kern
fn apply_operation(op: &fn(i32, i32) i32, a: i32, b: i32) i32 {
    return op(a, b);
}
```

空捕获闭包可以自然传给它：

```kern
let sum = apply_operation([](a: i32, b: i32) i32 {
    return a + b;
}, 10, 20);
```

`&fn(i32, i32) i32` 是 thin pointer，只保存代码地址。它没有闭包状态。

## 捕获必须显式

闭包语法是：

```kern
[captures](args) ReturnType {
    return value;
}
```

捕获列表里的名字是显式的：

```kern
let base = 100i32;

let add_base = [base](value: i32) i32 {
    return base + value;
};
```

如果要捕获一个指针，直接在捕获列表里命名绑定：

```kern
let mut counter = 0i32;

let bump = [ptr = counter..&]() void {
    ptr.* += 1;
};
```

这里 `ptr = counter..&` 捕获的是指向 `counter` 的可写指针。闭包体里修改的是 `ptr.*`，不是复制出来的整数。

捕获项右边是普通表达式，不是一套专门的捕获语言。也就是说，`name = expr` 里的 `expr` 可以是取地址、字段访问、函数调用，甚至是一个块表达式：

```kern
let mut counter = 0i32;

let bump = [ptr = {
    let p = counter..&;
    p
}]() void {
    ptr.* += 1;
};
```

日常写代码时，把 `&T` / `&mut T` 当成“读视图 / 写视图”来理解通常足够。但在 Kern 的语义里，指针本身仍然是一个普通值：闭包捕获的是这个值，之后闭包体再通过这个值访问目标存储。这个模型让闭包捕获、指针、块表达式和普通值初始化保持同一套规则。

## `&Fn` 和 `&mut Fn`

带状态的闭包要通过 closure fat pointer 调用：

```kern
fn process_with_context(cb: &Fn(i32) i32, value: i32) i32 {
    return cb(value);
}

let base = 100i32;
let result = process_with_context([base](value: i32) i32 {
    return base + value;
}, 23);
```

`&Fn(Args) Ret` 是只读调用接口；`&mut Fn(Args) Ret` 允许调用时修改捕获状态：

```kern
fn repeat_twice(cb: &mut Fn() void) void {
    cb();
    cb();
}

let mut counter = 0i32;
repeat_twice([ptr = counter..&]() void {
    ptr.* += 1;
});
```

当函数参数明确要求 `&Fn` 或 `&mut Fn` 时，Kern 可以把栈上的闭包状态自然包装成 fat pointer 传进去。这是边界自然转换的一部分。

## 闭包状态是一个值

写出闭包表达式时，得到的是一个具体的匿名状态值：

```kern
let left = 10i32;
let right = 20i32;

let closure = [left, right](x: i32) i32 {
    return (left + right) * x;
};
```

这个匿名类型不能直接写出名字，但可以通过 `@typeOf(closure)` 查询。它默认和普通局部值一样在当前作用域里。

如果上下文没有自动转换，也可以显式 cast 闭包状态指针：

```kern
let cb = closure.& as &Fn(i32) i32;
let out = cb(2);
```

这里传入的是闭包状态的指针。fat pointer 自己不是那个状态，只是“状态指针 + 调用入口”的动态接口。

## 逃逸闭包需要显式分配

不要返回指向栈上闭包状态的 `&Fn`。如果闭包要逃逸当前函数，必须把它的状态放到更长生命周期的存储里，例如 allocator 分配的内存。

```kern
use base.mem.{Allocator, Layout, layout_of};

struct StoredClosure {
    callback: &Fn(i32) i32,
    layout: Layout,
};

fn create_heap_closure(alloc: &mut Allocator, factor: i32) StoredClosure {
    let stack_closure = [factor](value: i32) i32 {
        return factor * value;
    };

    let layout = layout_of[@typeOf(stack_closure)]();
    let raw = match alloc.alloc(layout) {
        .{ Some: storage } => storage as &mut @typeOf(stack_closure),
        .None => @trap(),
    };

    raw.* = stack_closure;
    return .{
        callback: raw as &Fn(i32) i32,
        layout,
    };
}
```

释放时，closure fat pointer 不能假装成普通 struct 访问内部字段。用 `.@statePtr()` 提取它的状态指针：

```kern
defer alloc.free(stored.callback.@statePtr() as &mut u8, stored.layout);
```

`.@statePtr()` 和 `.@len()` 这类 member intrinsic 是编译器拥有的投影原语，不是普通方法。表示层投影应直接使用这种显式的 `.@name()` 写法，而不是通过普通库方法包装。

## 常用场景

闭包在 Kern 里经常用于：

- `Option.map`、`Result.map`、`and_then` 这类值转换。
- `List.retain`、`Map.for_each` 这类容器访问。
- 文件树遍历、环境变量遍历等 visitor API。
- 局部测试和一次性策略函数。
