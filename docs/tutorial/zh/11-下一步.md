## 想写普通 hosted 工具

重点看：

- [`std.io`](../../../library/std/io/init.rn)：标准输出、标准错误、`Printable`。
- [`std.fs`](../../../library/std/fs)：路径、文件、目录。
- [`std.proc`](../../../library/std/proc)：参数、进程信息、shell capture。
- [`std.env`](../../../library/std/env)：环境变量。
- [`std.time`](../../../library/std/time/init.rn)：时间和 sleep。

参考示例：

- [`examples/io_and_files.rn`](../../../examples/io_and_files.rn)
- [`examples/collections.rn`](../../../examples/collections.rn)

## 想写容器、解析器或库

重点看 `base`：

- [`base.coll`](../../../library/base/coll/init.rn)：`List`、`String`、`Map`、`Tree`、range、slice helper。
- [`base.mem.alloc`](../../../library/base/mem/alloc/init.rn)：allocator trait、GPA、arena。
- [`base.io`](../../../library/base/io/init.rn)：`Read`、`Write`、`Formatable` 和格式化。
- [`base.num`](../../../library/base/num/init.rn)：数字常量和解析。

生态中的大型包，比如 JSON 或 bit-level I/O 库，也应该保持同样的
`base` 边界：allocator 策略显式留在 API 里，借用和所有权在调用点可见，
发布前用测试覆盖主要工作流。

## 想写 kernel 或 freestanding 程序

重点看：

- [`runtime-architecture.md`](../../runtime-architecture.md)
- [`craft.md`](../../craft.md) 的 freestanding package 部分
- [`kernc.md`](../../kernc.md) 的 freestanding `_start` 和 linker script 部分
- [`examples/limine-smoke`](../../../examples/limine-smoke)

如果你在项目里使用自定义 `sys` 或 `rt`，把它们当成普通模块/包边界处理。不要假设编译器会为这些层提供隐藏 provider。
